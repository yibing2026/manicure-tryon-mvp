# 美甲 AI 试戴与智能运营系统提交说明

## 1. 项目概述

本项目面向美甲服务场景，围绕“用户看得见上手效果”和“运营实时感知流行趋势”两个核心痛点，构建了一个可本地运行、可演示、可迭代的 MVP 系统。

系统包含两条主线：

- AI 试戴：用户加载手部图片和美甲款式图后，可先通过本地画布生成试戴预览，再调用 OpenAI 或豆包图像生成 API 生成正式试戴效果图。
- 智能运营：基于官方款式数据建立初始风格标签、趋势分析和运营规则，通过运营 Copilot 页面实时生成款式推荐、活动文案、运营动作和风险提示。

## 2. 本地运行方式

进入项目目录：

```powershell
Set-Location D:\MVP
```

启动本地服务：

```powershell
npm start
```

浏览器打开：

```text
http://localhost:3000/index.html
http://localhost:3000/ops.html
```

页面说明：

- `index.html`：AI 美甲试戴 MVP 页面。
- `ops.html`：智能运营 Copilot 演示页面。

## 3. API 配置

项目支持 OpenAI 与豆包 / 火山方舟图像生成接口。请在 `D:\MVP\.env.local` 中配置密钥和模型参数。

示例字段：

```env
ARK_API_KEY=你的火山方舟 API Key
DOUBAO_IMAGE_MODEL=doubao-seedream-4-0-250828
DOUBAO_IMAGE_VERSION=
DOUBAO_IMAGE_RESPONSE_FORMAT=url
OFFICIAL_WORKBOOK_PATH=D:\Manicure\命题三美甲评测数据（对外版）.xlsx
```

如果使用 OpenAI：

```env
OPENAI_API_KEY=你的 OpenAI API Key
OPENAI_IMAGE_MODEL=gpt-image-1
```

注意：提交材料中不要包含真实 API Key。

## 4. 演示流程

### 4.1 AI 试戴演示

1. 打开 `http://localhost:3000/index.html`。
2. 在“官方样例一键加载”区域选择官方手图和款式图。
3. 点击“一键加载样例”。
4. 在画布上查看初始贴合预览。
5. 如有需要，使用“单指微调”调整甲面位置、大小和旋转角度。
6. 选择生成提供方，推荐使用“豆包 / 火山方舟”。
7. 点击“生成正式试戴图”。
8. 在页面下方查看模型生成结果。

### 4.2 智能运营演示

1. 打开 `http://localhost:3000/ops.html`。
2. 在左侧切换运营场景，例如“约会转化”“派对上新”“婚礼专题”“节日种草”。
3. 中间表单会自动填充人群、场景、候选款式和运营目标。
4. 点击“生成策略”。
5. 右侧查看推荐款式、匹配分、活动文案、运营动作、风险提示和接口返回 JSON。

## 5. 技术路线

### 5.1 AI 试戴

当前版本不训练独立图像分割模型，而是采用“手图 + 款式图 + 画布位置引导图”的多图输入方式，将试戴任务交给图像生成模型完成。

核心原因：

- 官方数据量较小，不适合从零训练稳定的分割或生成模型。
- 第一个比赛版本更需要快速验证用户体验闭环。
- 画布引导图可以给模型提供甲面位置、尺度和方向提示，降低纯文本控制的不确定性。

适用边界：

- 对姿态简单、指甲区域清晰的手图效果较好。
- 对遮挡严重、手指弯曲明显、甲面不可见的图片，仍需要人工微调或重新生成。

### 5.2 智能运营

当前运营模块采用“结构化标签 + 规则引擎 + Copilot 展示页”的方案。

核心流程：

1. 从官方评测表抽取款式数据。
2. 为 25 个官方款式建立初始标签，包括颜色、风格、场景、人群、价格带和趋势关键词。
3. 基于标签生成趋势摘要和运营规则。
4. 后端接口 `/api/ops-copilot-demo` 根据人群、场景、候选款式和趋势簇进行打分。
5. 前端运营页将结果展示为推荐款式、活动文案、运营动作和风险提示。

核心原因：

- 小数据场景下，规则系统比直接使用大模型更稳定、可解释、易评测。
- 后续可以保留相同接口协议，升级为“规则召回 + 大模型生成文案”的混合方案。

## 7. 已完成能力

- 手图与款式图一键加载。
- 本地画布生成甲面位置引导图。
- 支持 OpenAI / 豆包图像生成 API。
- API 调用日志，记录 requestId、provider、模型、耗时、失败原因和重试信息。
- 用户个性化推荐，根据肤色、手型、场景、风格偏好、预算、mock 热度和试戴质检结果推荐款式。

- 自动质检模块 v1，评估输出存在性、尺寸、场景漂移和指甲区域变化。
- 安全版 Agent Workflow v1，串联质检、重试计划、归档摘要和运营动作。
- 运营日报生成，汇总热门款式、增长预警、推荐池、复查队列、运营动作和风险提示。
- 官方 25 个款式初始标签。
- 款式趋势统计与运营摘要。
- 规则式运营策略生成接口。
- 可视化运营 Copilot 演示页面。

## 9. 评测与迭代方向

短期可评测指标：

- 试戴生成成功率。
- 甲面位置贴合度。
- 款式风格保真度。
- 用户从加载样例到看到结果的耗时。
- 运营推荐结果是否符合人群、场景和库存约束。

后续迭代方向：

- 增加自动甲面检测或轻量分割能力。
- 为生成图增加自动质量评分。
- 将当前安全版 Workflow 升级为“人工确认预算后自动重试并重新评估”。
- 接入真实点击、收藏、下单数据，动态更新热度规则。
- 将运营规则接口升级为“规则召回 + LLM 文案生成 + 策略解释”的混合 Copilot。
