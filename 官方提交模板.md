# 美甲 AI 试戴与智能运营 Agent 系统

## 1. 项目名称

美甲 AI 试戴与智能运营 Agent 系统

## 2. 项目简介

本项目面向美甲服务场景，围绕用户“看不见真实上手效果”和商家“难以及时识别趋势并制定运营策略”两个痛点，构建了一个可本地运行、可演示、可评测、可迭代的 MVP 系统。

系统包含用户侧 AI 试戴与商家侧智能运营两条主线，并通过个性化推荐、试戴质量评估和 Agent Workflow 将两者连接成闭环。

## 3. 赛题痛点理解

用户侧痛点：

- 用户浏览款式图时难以判断真实上手效果。
- 用户担心肤色、手型、甲长与款式不匹配。
- 到店试戴成本高，决策周期长，容易放弃预约。

运营侧痛点：

- 款式库存多，人工统计热度和风格趋势效率低。
- 热门款式识别滞后，错过运营窗口。
- 推荐策略粗放，无法根据人群、场景和试戴质量动态调整。

## 4. 解决方案

本项目采用“多模态生成 API + 个性化推荐 + 自动质检 + 智能运营 Copilot”的方案。

核心流程：

```text
用户手图
-> 自动估计肤色和手型
-> 用户选择场景、风格、甲长和预算
-> 个性化推荐款式 Top N
-> 加载推荐款式
-> 豆包 / OpenAI 生成正式试戴图
-> 自动质检 pass / review / fail
-> 生成重试计划与推荐池
-> 运营侧生成推荐策略、活动文案和风险提示
```

## 5. 系统模块

### 5.1 AI 试戴模块

- 支持本地上传手图和款式图。
- 支持官方评测表 URL 一键加载样例。
- 支持豆包 / 火山方舟和 OpenAI 图像生成接口。
- 可通过推荐款式直接调用豆包生成正式试戴图。
- 页面不展示粗糙大椭圆贴图，引导图仅在后台生成并传给模型。

### 5.2 用户个性化推荐模块

- 根据手图自动估计肤色和手型。
- 用户选择场景、偏好风格、甲长偏好和预算。
- 综合官方款式标签、mock 热度数据、试戴质检结果进行规则打分。
- 输出推荐款式、推荐分、推荐理由和风险提示。
- 支持一键加载推荐款式并生成正式试戴图。

### 5.3 自动质检模块

- 对生成结果进行规则版质量评估。
- 检查输出是否存在、尺寸是否异常、场景是否漂移、指甲区域变化是否过弱。
- 输出 pass / review / fail 队列。
- 当前官方样例评估结果：13 个样例中 11 个 pass，2 个 review，0 个 fail。

### 5.4 Agent Workflow 模块

- 读取已有试戴结果。
- 自动运行质检。
- 根据质检结果生成重试计划。
- 将 pass 样例进入推荐池，review / fail 样例进入复查队列。
- 生成运营动作建议。

### 5.5 智能运营 Copilot 模块

- 基于官方 25 个款式建立标签体系。
- 根据人群、场景、候选款式和趋势规则进行打分。
- 输出推荐款式、活动文案、运营动作和风险提示。
- 适合作为商家运营后台的决策辅助工具。

## 6. 数据说明

已使用数据：

- 官方评测表中的手图 URL 和款式图 URL。
- 官方 25 个款式的结构化标签。
- 官方 13 组配对样例的生成结果。
- 自动质检报告。

模拟数据：

- `data/mock_style_popularity.json`
- 用于模拟曝光、试戴、收藏、预约等热度信号。
- 当前仅用于验证推荐闭环，不代表真实用户行为数据。

后续可接入真实数据：

- 曝光量。
- 点击量。
- 试戴次数。
- 收藏数。
- 预约数。
- 成交数。
- 退款或差评数据。

## 7. 技术路线

前端：

- 原生 HTML / CSS / JavaScript。
- Canvas 用于手图展示和隐藏引导图生成。
- 用户侧试戴页与商家侧运营页分离展示。

后端：

- Node.js HTTP 服务。
- 提供官方样例读取、图像生成、个性化推荐、运营 Copilot 等接口。

图像生成：

- 豆包 / 火山方舟图像生成 API。
- OpenAI 图像生成接口作为可选方案。

数据分析与 Agent Workflow：

- Python 脚本生成款式标签、运营摘要、质检报告和 Agent Workflow 报告。

## 9. 本地运行方式

进入项目目录：

```powershell
Set-Location D:\MVP
```

启动服务：

```powershell
$env:PORT="3101"
node server.mjs
```

打开页面：

```text
http://localhost:3101/index.html
http://localhost:3101/ops.html
```

非付费验证：

```powershell
npm run evaluate:tryon
npm run workflow:tryon
```

## 10. 演示流程

用户侧演示：

1. 打开试戴页。
2. 一键加载官方手图。
3. 系统自动估计肤色和手型。
4. 用户选择场景、风格、甲长和预算。
5. 点击生成推荐款式。
6. 选择推荐款式并点击豆包生成试戴图。
7. 查看正式试戴结果。

运营侧演示：

1. 打开运营 Copilot 页面。
2. 选择运营场景。
3. 点击生成策略。
4. 查看推荐款式、活动文案、运营动作和风险提示。
5. 查看 Agent Workflow 报告中的推荐池和复查队列。

## 11. 创新点

- 将 AI 试戴与智能运营从两个独立功能连接为闭环。
- 使用用户画像和款式标签实现可解释推荐。
- 使用隐藏引导图辅助图像生成，避免用户界面出现粗糙贴图。
- 在小数据场景下采用规则系统和 mock 热度验证推荐闭环。
- 引入自动质检和 Agent Workflow，让系统具备可评测、可复查、可迭代能力。

## 12. 当前边界

- 当前没有接入真实用户行为日志。
- mock 热度数据仅用于 MVP 验证。
- 肤色和手型估计是轻量规则版，不是专业视觉识别模型。
- 自动质检 v1 不能精确判断所有指甲贴合偏移问题。
- 图像生成质量依赖外部多模态生成模型。

## 13. 后续迭代

- 接入真实曝光、点击、收藏、预约和成交数据。
- 使用手部关键点或视觉模型优化肤色、手型和指甲区域识别。
- 将 review / fail 样例自动重试并重新评估。
- 将规则运营 Copilot 升级为“规则召回 + LLM 文案生成 + 策略解释”。
- 部署在线 Demo 并补充演示视频。
